# 南航上海路校园生活小站项目规则

本文件适用于项目根目录及其所有子目录。Codex 或其他自动化开发工具处理本项目时，必须遵守以下规则。

## 技术栈与项目结构

1. 本项目后端使用 Java、Spring Boot、MyBatis 和 MySQL，前端使用原生 HTML、CSS 和 JavaScript。
2. 不要引入 Vue、React、Node.js 构建工具或其他前端框架，除非用户明确要求。
3. 不要修改后端接口路径，除非用户明确要求。
4. 不要删除 `product`、`service_item` 等核心表及其相关业务逻辑。
5. 不要重构项目结构，除非用户明确要求。

## 前端规则

1. 前端线上部署时，后端 API 地址为：

   ```text
   http://115.159.47.131:8080
   ```

2. 本地开发时，后端 API 地址可以为：

   ```text
   http://localhost:8080
   ```

3. 修改 API 地址时，必须说明该修改对本地开发和线上部署的影响。
4. 前端页面应保留或补充清楚的中文注释，内容要适合 Java 和前端新手理解。

## 安全与敏感信息

1. 不要把真实数据库密码、服务器密码、Token 或其他敏感凭据写入 README、`DEPLOYMENT.md`、代码或 Git 提交记录。
2. 部署文档中的数据库密码统一使用以下占位符：

   ```text
   <DB_PASSWORD>
   ```

3. 服务器部署相关操作应优先使用安全、可解释的命令。
4. 不要执行删除数据库、删除核心表、清空正式数据、向公网暴露 MySQL `3306` 端口等高风险操作。
5. 涉及线上部署或服务重启时，应优先提醒用户当前线上服务可能受到影响。

## 修改与说明流程

1. 修改代码或文档前，先说明准备修改哪些文件。
2. 修改完成后，说明实际修改了哪些文件，以及每个文件的主要变化。
3. 每次功能修改后，应尽量运行相关测试；如果无法运行，应明确说明测试方法、未测试原因和可能风险。
4. 修改应保持最小范围，不要增加无关功能或顺手重构无关代码。

## Git 操作规则

1. 涉及 Git 操作时，先执行 `git status`，确认当前分支和工作区修改。
2. 根据 `git status` 的结果决定需要暂存的文件，不要误提交用户已有的无关修改。
3. 执行 `git add`、`git commit` 或 `git push` 前，应明确本次操作包含的文件和内容。
4. 如果推送失败，不要重复创建相同提交；应先判断远程地址、网络和代理配置。
